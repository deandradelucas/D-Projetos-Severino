================================================================================
Severino — API no EasyPanel (Docker)
================================================================================
O repositório tem Dockerfile na raiz: só a API Node (Hono).

Produção típica (Vercel / mesmo site): front + API em
https://severino.mestredamente.com — pedidos /api na mesma origem.

API só em Docker na VPS/EasyPanel (subdomínio dedicado): exemplo
https://api.severino.mestredamente.com — aí o build do front precisa
VITE_SEVERINO_API_ORIGIN apontando para esse host (ou proxy /api no Nginx para o mesmo domínio severino…).

1) EasyPanel — novo serviço / app
   - Tipo: **Docker** (build a partir do Git ou upload).
   - Repositório: **D-Projetos-Severino**, branch **main**.
   - Dockerfile: caminho **Dockerfile** (raiz do repo).
   - **Porta interna exposta:** **3001** (é o API_PORT do container).

2) Variáveis de ambiente (no painel do serviço — não commits!)
   Copiar do teu env.example / produção, no mínimo:
   - SUPABASE_URL  (ou VITE_SUPABASE_URL — o servidor aceita ambos no código admin)
   - SUPABASE_SERVICE_ROLE_KEY
   - (opcional) GEMINI_API_KEY, ASAAS_API_KEY, ASAAS_API_BASE_URL, ASAAS_WEBHOOK_TOKEN, etc.

   Se o EasyPanel injeta **PORT** diferente de 3001, define também:
   - API_PORT=<mesmo valor que o proxy espera>
   - API_HOST=0.0.0.0   (já é o default na imagem)

3) Domínio / SSL no EasyPanel
   - Adiciona hostname: **api.severino.mestredamente.com** (API isolada) OU usa só **severino.mestredamente.com** se o proxy /api for no mesmo stack.
   - SSL automático (Let’s Encrypt) pelo próprio EasyPanel.
   - Na zona DNS do domínio: **A** ou **CNAME** para o servidor onde corre o EasyPanel
     (conforme o painel indica — pode ser IP do VPS ou target da CDN).

4) Health check (opcional no EasyPanel)
   - HTTP GET **/api/health**
   - Esperado: JSON com "ok": true

5) Deploy / rebuild
   - Cada push ao **main** pode disparar rebuild conforme configurares webhooks.
   - Depois do deploy: abre no browser
     https://api.severino.mestredamente.com/api/health

6) Front / bundle
   - Se o utilizador abre **severino.mestredamente.com** e a API está no mesmo host (/api), não forces api.severino no bundle.
   - API noutro host: define **VITE_SEVERINO_API_ORIGIN** no build com esse URL (sem barra final).

Resolução de problemas:
   - 502/503 no domínio: serviço parado, porta errada, ou health falhou no deploy.
   - CORS: origem https://severino.mestredamente.com já é aceite pelo servidor (*.mestredamente.com).
