# Story 7.5 — API Route + Web UI de Import

## Status
Draft

## Descrição
Criar rota `POST /api/import/planilha` que aceita upload multipart de arquivo Excel ou PDF, e um componente React de upload na página de Transações (ou em página dedicada) com preview do resultado e confirmação antes de salvar. Esta story entrega a experiência de import via interface web — complementar ao WhatsApp (Story 7.4).

## Critérios de Aceite

### Backend — Rota de Import
- [ ] Nova rota em `server/routes/register-import.mjs` (novo arquivo)
- [ ] `POST /api/import/planilha` — multipart/form-data, campo `arquivo`
- [ ] Autenticação via `parseUsuarioEscopoApi` (mesmo padrão das outras rotas)
- [ ] Limite de tamanho: rejeita > 20MB com 413
- [ ] Validação de mimeType (xlsx, xls, csv, pdf, ofx, qfx) — rejeita outros com 400
- [ ] Aceita também por extensão do filename quando mimeType for `application/octet-stream`
- [ ] Rate limit: 5 imports por hora por usuário (`import-planilha:{actorId}`)
- [ ] Rota registrada em `server/routes/register-all.mjs`

### Backend — Endpoint de Preview (opcional, fase 2)
- [ ] `POST /api/import/preview` — mesmo upload, retorna as primeiras 10 transações parsed SEM salvar
- [ ] Útil para o usuário conferir antes de confirmar
- [ ] **Opcional para esta story** — pode ser implementado junto com UI ou deixado para sprint futura

### Frontend — Componente de Upload
- [ ] Componente `ImportarPlanilhaModal.jsx` em `src/components/transacoes/`
- [ ] Botão de trigger: "📊 Importar Planilha" na página de Transações (toolbar superior)
- [ ] Modal com:
  - Área de drag-and-drop ou click para selecionar arquivo
  - Instruções: "Envie seu extrato bancário em Excel (.xlsx, .xls, .csv), PDF ou OFX (.ofx, .qfx)"
  - Formatos aceitos visíveis no modal: `.xlsx .xls .csv .pdf .ofx .qfx`
  - Dica contextual: "OFX: exportado diretamente pelo seu banco (Bradesco, Itaú, BB, Caixa...)"
- [ ] Ao selecionar arquivo: valida tamanho e tipo no cliente antes de enviar
- [ ] `accept` do input: `.xlsx,.xls,.csv,.pdf,.ofx,.qfx`
- [ ] Loading state durante upload/processamento
- [ ] Resultado exibido no modal:
  ```
  ✅ 47 transações importadas
  40 despesas · 7 receitas
  2 categorizadas como "Outros" (revise no app)
  ```
- [ ] Botão "Fechar" — fecha modal e atualiza lista de transações (trigger refetch)
- [ ] Erros exibidos inline no modal (não toast) com mensagem clara

### Frontend — Integração na Página de Transações
- [ ] Botão "Importar" visível na barra de ações da página de Transações
- [ ] Após import bem-sucedido: refetch automático da lista de transações
- [ ] Dark mode: modal segue `var(--bg-card)`, bordas, texto padrão do design system

### Tratamento de Erros
- [ ] 413 (arquivo muito grande): "Arquivo muito grande. Máximo: Excel 10MB / PDF 20MB."
- [ ] 400 (formato inválido): "Formato não suportado. Use .xlsx, .xls, .csv ou .pdf."
- [ ] 429 (rate limit): "Você já importou várias planilhas hoje. Tente novamente amanhã."
- [ ] 5xx: "Erro ao processar o arquivo. Tente novamente."

## Notas Técnicas

**Hono multipart:**
```js
const body = await c.req.parseBody()
const file = body['arquivo']  // File object
const buffer = Buffer.from(await file.arrayBuffer())
const mimeType = file.type
```

**Frontend fetch multipart:**
```js
const formData = new FormData()
formData.append('arquivo', file)
const res = await fetch('/api/import/planilha', { method: 'POST', body: formData, headers: { 'x-user-id': userId } })
```

**Hono já suporta multipart nativo** — não precisa de middleware extra
