# Story 2.1: Design System Audit & Token Foundation

## Status
Ready

## Descrição
Auditar todo o frontend atual para mapear inconsistências visuais (valores hardcoded, tokens não-canônicos, espaçamentos avulsos, tipografia inconsistente) e expandir o `@theme` de `src/index.css` com os tokens completos que servirão de fundação para todas as stories do epic 2.

## Critérios de Aceite
- [ ] Relatório de auditoria gerado em `docs/design-system-audit.md` listando todas as inconsistências encontradas por categoria (cor, tipografia, espaçamento, sombra, border-radius, motion)
- [ ] `src/index.css` expandido com tokens canônicos de: espaçamento (scale 4/8/12/16/20/24/32/40/48/64px), tipografia (size + weight), border-radius (sm/md/lg/xl/full), elevation (shadow-sm/md/lg/gold) e motion (duration-fast/base/slow + easing-standard/decelerate)
- [ ] Guia de componentes criado em `docs/component-guide.md` documentando: como usar cada token, padrão de card, padrão de botão primário/secundário/ghost, padrão de input, padrão de lista de transações
- [ ] Zero valores hex/px hardcoded introduzidos nos arquivos modificados
- [ ] `npm run lint` passa sem erros

## Escopo
**IN:**
- Leitura de todas as páginas em `src/pages/` e componentes em `src/components/`
- Edição de `src/index.css` (bloco @theme)
- Criação de `docs/design-system-audit.md`
- Criação de `docs/component-guide.md`

**OUT:**
- Nenhuma alteração em JSX de páginas ou componentes nesta story
- Nenhuma alteração em `dashboard.css` (vem na 2.2)
- Backend/API intocados

## Dependências
- Story 1.2 concluída (estabilidade do backend garantida)
- Leitura do `docs/theme-guidelines.md` existente

## Complexidade
M (médio) — análise intensiva, sem implementação de UI

## Valor de Negócio
Fundação de todas as demais stories. Sem ela, as melhorias visuais serão inconsistentes e retrabalho é certo.

## Riscos
- Tokens existentes podem ser usados de forma diferente em cada página — mapear tudo antes de consolidar
- Não sobrescrever tokens que o `dashboard.css` referencia sem coordenar com Story 2.2
