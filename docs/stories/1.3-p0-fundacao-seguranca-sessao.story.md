## Estimativa
**8 pontos** — Ops SQL + backend guard + varredura front em várias páginas.

## Valor de Negócio
Reduz risco de vazamento via PostgREST, evita acesso indevido em falha de DB no gate de assinatura, e elimina sessão “morta” após F5 — principal causa de suporte “não carrega transações”.

## Riscos

| Risco | Mitigação |
|-------|-----------|
| RLS quebra integração se algo usar anon key nas tabelas core | Deny-by-default; API usa só `service_role`; smoke pós-apply |
| Migration 19 em DB grande | Índices `IF NOT EXISTS`; aplicar em janela de baixo tráfego |
| Regressão em páginas migradas para `apiFetch` | Test manual por hub; lint/build CI |
