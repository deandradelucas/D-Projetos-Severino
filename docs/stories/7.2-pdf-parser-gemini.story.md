# Story 7.2 — PDF Parser via Gemini Inline

## Status
Draft

## Descrição
Criar `server/lib/import/pdf-parser.mjs` que recebe um Buffer de arquivo PDF (extrato bancário), converte para base64 e envia ao Gemini usando `inlineData` (suporte nativo a PDF do Gemini 1.5). O Gemini extrai todas as transações do extrato e retorna um array normalizado de transações — sem necessidade de nenhuma biblioteca de parsing de PDF.

## Critérios de Aceite

### Parsing via Gemini Inline
- [ ] Função exportada: `async function parsePdfTransactions(pdfBuffer): Promise<ParsedRows | ErrorResult>`
- [ ] Converte buffer para base64 string
- [ ] Monta request Gemini com `inlineData: { mimeType: 'application/pdf', data: base64 }`
- [ ] Usa modelo `gemini-1.5-flash` (suporte a PDF; flash é mais rápido/barato que pro)
- [ ] `maxOutputTokens: 4096` (extratos longos podem ter muitas transações)
- [ ] `temperature: 0.1` (determinístico)

### Prompt de Extração
- [ ] Prompt instrui Gemini a extrair TODAS as transações do extrato
- [ ] Output esperado: JSON array `[{data, descricao, valor, tipo}]`
  - `data`: formato `YYYY-MM-DD`
  - `descricao`: string sem quebras de linha, max 255 chars
  - `valor`: número positivo
  - `tipo`: `"DESPESA"` ou `"RECEITA"`
- [ ] Prompt instrui a ignorar totais, saldos e linhas de cabeçalho
- [ ] Prompt inclui exemplo explícito de output esperado (few-shot)

### Normalização e Validação
- [ ] Response do Gemini é parseado via `tryParseJsonBlock` (já existe em `server/lib/ai/parsers.mjs`)
- [ ] Cada item é validado: data válida, valor > 0, descricao não vazia, tipo válido
- [ ] Itens inválidos são silenciosamente descartados (não jogam erro)
- [ ] Retorna `Array<{data: string, descricao: string, valor: number, tipo: 'DESPESA'|'RECEITA'}>`

### Tratamento de Erros e Limites
- [ ] Arquivo > 20MB retorna `{ error: 'ARQUIVO_MUITO_GRANDE', message: 'PDFs até 20MB são suportados.' }`
- [ ] Gemini retorna 0 transações → `{ error: 'NENHUMA_TRANSACAO', message: 'Não encontrei transações no PDF. Tente um extrato bancário padrão.' }`
- [ ] Falha de API Gemini → `{ error: 'FALHA_IA', message: 'Não consegui processar o PDF agora. Tente novamente.' }`
- [ ] PDF corrompido ou não-PDF → captura erro de parsing e retorna `{ error: 'FORMATO_INVALIDO' }`

### Fallback de Modelo
- [ ] Tenta `gemini-1.5-flash` primeiro
- [ ] Se falhar, tenta `gemini-1.5-pro` (usa `resolveGeminiModelCandidates` existente)
- [ ] Se ambos falharem, retorna erro `FALHA_IA`

## Escopo

## Notas Técnicas

**Prompt exemplo:**
```
Você é um extrator de transações financeiras. Analise este extrato bancário e extraia TODAS as transações.

Retorne APENAS um array JSON válido (sem markdown) no formato:
[
  {"data":"2024-01-15","descricao":"MERCADO EXTRA","valor":87.50,"tipo":"DESPESA"},
  {"data":"2024-01-20","descricao":"SALARIO EMPRESA XYZ","valor":3500.00,"tipo":"RECEITA"}
]

Regras:
- Ignore totais, saldos, cabeçalhos e rodapés
- valor sempre positivo
- tipo: "DESPESA" para saídas/débitos, "RECEITA" para entradas/créditos
- data no formato YYYY-MM-DD
- Se não encontrar transações, retorne []
```

**Limitação de tamanho Gemini:**
- Gemini 1.5 Flash: até 1048576 tokens de contexto; PDF de ~20MB gera ~500K tokens
- PDFs de extrato bancário típicos: 1-5MB → bem dentro do limite
