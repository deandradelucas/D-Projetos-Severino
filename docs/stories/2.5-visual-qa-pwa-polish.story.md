# Story 2.5: Visual QA & PWA Polish

## Status
Ready

## Descrição
QA visual completo do epic 2, polish final e validação PWA. Esta story é o gate de qualidade antes de o produto ser considerado "pronto para vender". Nada de subjetividade — checklists objetivos, itens com localização exata, severidade definida.

## Critérios de Aceite

### QA Visual (executado pelo @visual-qa)
- [ ] Relatório gerado em `docs/qa/visual-qa-epic2.md` com todos os itens verificados
- [ ] Zero itens HIGH em aberto no relatório final
- [ ] Todos os itens MEDIUM documentados com plano de resolução ou aceite consciente

### Consistência de Design System
- [ ] Auditoria de tokens: nenhum hex/px hardcoded em arquivos modificados pelo epic 2
- [ ] Tipografia consistente em todas as páginas (family, size, weight, line-height)
- [ ] Espaçamentos seguem escala definida na Story 2.1
- [ ] Border-radius idêntico entre cards, modais e inputs
- [ ] Sombras usando exclusivamente tokens --shadow-*

### Mobile (375px — iPhone SE)
- [ ] Nenhum overflow horizontal em nenhuma página
- [ ] MobileBottomNav não cobre conteúdo crítico
- [ ] Formulários completamente usáveis com teclado virtual aberto
- [ ] Todos os CTA com área de toque ≥ 44px

### Contraste e Acessibilidade mínima
- [ ] Texto primário sobre fundo: contraste ≥ 4.5:1 (WCAG AA)
- [ ] Texto de valor financeiro (KPI): contraste ≥ 7:1 (WCAG AAA)
- [ ] Elementos de input com label visível ou aria-label

### PWA Polish
- [ ] Manifest revisado: `name`, `short_name`, `description`, `start_url`, `display`, `theme_color`, `background_color` corretos
- [ ] Ícones 192x192 e 512x512 maskable renderizando corretamente
- [ ] Splash screen consistente com o visual do app (background #050607)
- [ ] Service worker registrado sem erro no console
- [ ] Offline: página de fallback adequada (não tela branca)
- [ ] Install prompt (`PwaInstallPrompt.jsx`) funcionando em Chrome Android e Safari iOS

### Performance básica
- [ ] Nenhuma imagem não-otimizada adicionada pelo epic 2
- [ ] CSS do epic 2 não adiciona mais de 5kb ao bundle final
- [ ] Nenhum re-render desnecessário introduzido (verificar React DevTools em Dashboard)

### Testes
- [ ] `npm run lint` passa sem warnings novos
- [ ] `npm run test` passa (unit + build + lint)
- [ ] `npm run audit:dashboard-css` sem classes órfãs novas
