# Story 2.3: Páginas Core — Refinamento Visual

## Status
Ready

## Descrição
Aplicar o design system definido na Story 2.1 em todas as páginas de conteúdo. O objetivo é consistência total: o usuário deve sentir que cada tela pertence ao mesmo produto premium. Foco nas páginas de alto uso diário (Dashboard, Transações, Relatórios) e páginas de primeira impressão (Login, Cadastro, Pagamento).

## Critérios de Aceite

### Dashboard
- [ ] Cards KPI com elevação consistente (`--shadow-gold` no card principal de saldo)
- [ ] Saldo exibido com tipografia `--font-size-2xl` ou maior, peso bold
- [ ] Lista de transações recentes com separador sutil (`--color-border`) e ícone de categoria padronizado
- [ ] Empty state ilustrado e encorajador quando não há transações
- [ ] Skeleton screens idênticos ao layout final (mesmo tamanho de cada elemento)

### Transações
- [ ] Cabeçalho de filtros padronizado com chips usando tokens (border, bg, texto)
- [ ] Linha de transação: ícone + nome + categoria + valor alinhados consistentemente
- [ ] Valores positivos em `--color-success`, negativos em `--color-error`
- [ ] "Carregar mais" como botão ghost estilizado, não link simples
- [ ] Empty state com mensagem clara e CTA para nova transação

### Relatórios
- [ ] Gráficos com cores usando tokens da paleta (accent-gold para receita, error para despesa)
- [ ] Cards de resumo com tipografia e espaçamento consistentes
- [ ] Seletor de período padronizado com estilo do design system

### Agenda
- [ ] Itens de agenda com mesmo padrão visual da lista de transações
- [ ] Status de evento com badge usando tokens de cor

### Login / Cadastro
- [ ] Formulários com inputs padronizados: border `--color-border`, focus ring `--color-accent-gold`
- [ ] Botão primário com efeito gold e border-radius consistente
- [ ] Logotipo/marca centralizado com tamanho adequado para splash de PWA

### Configurações
- [ ] Seções com separador e títulos em `--color-text-secondary`
- [ ] Toggle de tema com animação suave
- [ ] Items de config com padding consistente e hover state

### Pagamento / BemVindoAssinatura
- [ ] Hierarquia visual clara: preço dominante, benefícios legíveis
- [ ] CTA principal em gold com hover animado
