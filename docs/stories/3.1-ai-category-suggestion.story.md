# Story 3.1: Sugestão Automática de Categoria por IA no Formulário de Transação

## Status
InReview

## Descrição
Quando o usuário digita uma descrição no campo "Descrição" do modal de nova transação, o sistema deve sugerir automaticamente a categoria e subcategoria mais adequadas usando heurísticas locais (sem custo de API) e Gemini como fallback. A sugestão preenche os campos apenas se ainda estiverem vazios, preservando a escolha manual do usuário. Um badge "✨ sugerida por IA" indica quando a categoria foi preenchida automaticamente.

## Critérios de Aceite

### Backend — Endpoint de Sugestão
- [x] Endpoint `POST /api/ai/suggest-category` criado em `server/routes/register-ai.mjs`
- [x] Body aceita `{descricao: string, tipo: "DESPESA"|"RECEITA"}`
- [x] Header `x-user-id` obrigatório com validação via `parseUsuarioEscopoApi`
- [x] Rate limit: 60 requisições/minuto por usuário (`ai-suggest-cat:{actorId}`)
- [x] Heurísticas locais (`enriquecerCategoriaPorTexto`) executadas **antes** de qualquer chamada Gemini
- [x] Gemini chamado **apenas** se heurísticas retornarem `categoria_id: null`
- [x] Prompt Gemini usa apenas categorias do tipo correto (DESPESA ou RECEITA)
- [x] IDs retornados pela Gemini são **validados** contra as categorias reais do usuário antes de retornar
- [x] Resposta sempre `{categoria_id: string|null, subcategoria_id: string|null}` — nunca lança erro 5xx (retorna nulls em caso de falha)
- [x] Função `suggestCategoryForTransaction(descricao, tipo, categoriasUsuario)` exportada de `server/lib/ai.mjs`

### Frontend — Debounce e Auto-preenchimento
- [x] Debounce de 600ms no campo `descricao` do `TransactionModal`
- [x] Sugestão disparada apenas se `descricao.trim().length >= 3`
- [x] Sugestão **não** é disparada se `formData.categoria_id` já estiver preenchido pelo usuário
- [x] Quando sugestão retorna `categoria_id`, o `formData` é atualizado via `setFormData` callback (sem race condition)
- [x] `setFormData` callback verifica novamente se `categoria_id` ainda está vazio antes de aplicar
- [x] Estado `aiSuggestedCat` (boolean) rastreia se a categoria atual foi sugerida por IA
- [x] `aiSuggestedCat` é resetado para `false` quando usuário muda `categoria_id` manualmente
- [x] `aiSuggestedCat` é resetado ao trocar o tipo (DESPESA ↔ RECEITA)
- [x] Badge "✨ sugerida por IA" exibido abaixo do selector de categoria quando `aiSuggestedCat === true`
- [x] Badge desaparece quando usuário muda a categoria manualmente
- [x] Indicador de loading (spinner discreto) visível enquanto `aiSuggesting === true`
- [x] Falhas silenciosas — erros de rede não mostram toast nem quebram o formulário

### Comportamento Geral
- [x] Modo edição (`editingTransaction !== null`) **não** dispara sugestão automática — categoria já existe
- [x] Quando tipo muda (DESPESA ↔ RECEITA), timeout pendente é cancelado e `aiSuggestedCat` é resetado
- [x] Quando modal fecha, todos os timeouts são limpos (sem memory leak)
- [x] Quando description é apagada (`descricao === ''`), sugestão não é disparada

## Estimativa
**3 pontos** — Feature bem delimitada, reutiliza infra existente, zero migração de banco.

## Valor de Negócio
Reduz fricção no cadastro de transações — usuário não precisa navegar no dropdown de categorias para casos comuns. Melhora taxa de categorização (menos transações sem categoria). Custo de API Gemini próximo de zero pela maioria das sugestões ser resolvida por heurísticas locais.

## Riscos
- **Baixo:** Gemini retornar UUID inválido → mitigado por validação antes de retornar
- **Baixo:** Latência da sugestão > 600ms → usuário já vê loading, form continua funcional
- **Mínimo:** Race condition entre debounce e submit → `setFormData` callback resolve

## Arquitetura de Implementação

### Backend — `server/lib/ai.mjs`
```
suggestCategoryForTransaction(descricao, tipo, categoriasUsuario):
  1. Normaliza descricao (trim, min 2 chars)
  2. Chama enriquecerCategoriaPorTexto(descricao, {tipo, categoria_id: null, subcategoria_id: null}, categoriasUsuario)
  3. Se categoria_id encontrado → retorna imediatamente (sem Gemini)
  4. Se GEMINI_API_KEY ausente → retorna {null, null}
  5. Monta prompt compacto com categorias filtradas por tipo
  6. Chama Gemini (gemini-2.5-flash, maxOutputTokens: 100, temperature: 0.1)
  7. Valida UUIDs retornados contra categorias reais
  8. Retorna {categoria_id, subcategoria_id} validados
```

### Backend — `server/routes/register-ai.mjs`
```
POST /api/ai/suggest-category
  Headers: x-user-id
  Body: {descricao, tipo}
  → parseUsuarioEscopoApi
  → rateLimitTake (60/min)
  → getCategorias(usuarioId)
  → suggestCategoryForTransaction(descricao, tipo, categorias)
  → return {categoria_id, subcategoria_id}
  → catch: return {categoria_id: null, subcategoria_id: null} (nunca 5xx)
```

### Frontend — `TransactionModal.jsx`
```
Estados adicionais:
  aiSuggesting: boolean (spinner)
  aiSuggestedCat: boolean (badge)
  suggestTimeoutRef: useRef(null) (debounce)

useEffect([formData.descricao, formData.tipo, isOpen]):
  - Cancela timeout anterior
  - Skip se isEditMode || desc.length < 3 || categoria_id já definido
  - setTimeout(600ms) → fetch /api/ai/suggest-category
  - setFormData(prev => prev.categoria_id ? prev : {...prev, categoria_id, subcategoria_id})
  - setAiSuggestedCat(true)
  - cleanup → clearTimeout

handleChangeWithAIReset (wrapper de handleChange):
  - Se name === 'categoria_id' → setAiSuggestedCat(false)
  - Chama handleChange(e)

handleTypeChange override:
  - Chama handleTypeChange original
  - setAiSuggestedCat(false)

Badge JSX (abaixo do CategorySelector de categoria):
  {aiSuggestedCat && (
    <p className="ai-suggested-hint">✨ Categoria sugerida pela IA</p>
  )}
  {aiSuggesting && <span className="ai-suggest-spinner" />}
```
