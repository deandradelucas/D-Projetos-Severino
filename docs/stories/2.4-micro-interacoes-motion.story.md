# Story 2.4: Micro-interações & Motion System

## Status
Ready

## Descrição
Implementar o sistema de movimento definido pelo UX Director usando os tokens de motion do design system. Micro-interações bem executadas transformam um app funcional em um produto premium que o usuário quer mostrar para alguém.

## Critérios de Aceite

### Entradas de tela
- [ ] Todas as páginas autenticadas com fade-in + translateY(8px→0) ao montar, 250ms ease-out
- [ ] Implementado via CSS `@keyframes` ou classe utilitária — não via JS animation library externa

### Cards e elementos interativos
- [ ] Cards KPI: `scale(1.01)` + shadow elevation no hover, 150ms ease
- [ ] Linhas de transação: highlight sutil (bg opacity) no hover, 150ms
- [ ] Botão primário gold: efeito ripple ao clicar (pseudo-elemento radial), 300ms

### Navegação
- [ ] Transição entre rotas: fade 200ms (sem slide horizontal — causa motion sickness em financeiro)
- [ ] Item ativo no nav com animação de entrada do indicador gold

### Modais e overlays
- [ ] TransactionModal: slide-up 300ms ease-decelerate + backdrop blur animado
- [ ] ConfirmDialog: scale(0.96→1) + fade, 200ms
- [ ] Toast: slide-in da direita, 250ms; auto-dismiss com fade, 200ms

### Dados e loading
- [ ] KPI numbers: contagem animada de 0 ao valor real ao carregar (1s, ease-out) usando CSS counter ou requestAnimationFrame simples
- [ ] Skeleton → conteúdo: crossfade 200ms sem layout shift
- [ ] Refresh indicator (revalidating): spinner gold sutil no header

### PWA Install Prompt
- [ ] `PwaInstallPrompt.jsx`: slide-up suave com dismiss animado

## Escopo

**OUT:**

- Nenhuma animação no Shell Hub (contrato estável)

## Dependências
- Story 2.3 concluída (páginas com design system aplicado)

## Complexidade
M (médio) — CSS bem estruturado, sem novas dependências

## Valor de Negócio
Motion bem feito é o diferencial entre "app funcional" e "produto premium". É o que faz o usuário falar "esse app é bonito" para alguém.

## Riscos
- `prefers-reduced-motion` deve ser respeitado — envolver todos os keyframes em `@media (prefers-reduced-motion: no-preference)`
- Animações muito pesadas em dispositivos antigos — testar em hardware real ou throttling DevTools
