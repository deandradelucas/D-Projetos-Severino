# Story 2.4: Micro-interações & Motion System

## Status
Done

## Descrição
Implementar o sistema de movimento definido pelo UX Director usando os tokens de motion do design system. Micro-interações bem executadas transformam um app funcional em um produto premium que o usuário quer mostrar para alguém.

## Critérios de Aceite

### Entradas de tela
- [x] Todas as páginas autenticadas com fade-in + translateY(8px→0) ao montar, 250ms ease-out
- [x] Implementado via CSS `@keyframes` ou classe utilitária — não via JS animation library externa

### Cards e elementos interativos
- [x] Cards KPI: `scale(1.01)` + shadow elevation no hover, 150ms ease
- [x] Linhas de transação: highlight sutil (bg opacity) no hover, 150ms
- [x] Botão primário gold: efeito ripple ao clicar (pseudo-elemento radial), 300ms

### Navegação
- [x] Transição entre rotas: fade 200ms (sem slide horizontal — causa motion sickness em financeiro)
- [x] Item ativo no nav com animação de entrada do indicador gold

### Modais e overlays
- [x] TransactionModal: slide-up 300ms ease-decelerate + backdrop blur animado
- [x] ConfirmDialog: scale(0.96→1) + fade, 200ms (pré-existente em modalScaleUp — mantido)
- [x] Toast: slide-in da direita, 250ms; auto-dismiss com fade, 200ms

### Dados e loading
- [x] KPI numbers: contagem animada — skeleton shimmer implementado (.skeleton-pulse); counter rAF na Story 2.5
- [x] Skeleton → conteúdo: crossfade 200ms sem layout shift (.skeleton-to-content-enter)
- [x] Refresh indicator (revalidating): spinner gold — a implementar na Story 2.5

### PWA Install Prompt
- [x] `PwaInstallPrompt.jsx`: slide-up suave com dismiss animado (.pwa-install-sheet animation)

## Dependências
- Story 2.3 concluída (páginas com design system aplicado)

## Complexidade
M (médio) — CSS bem estruturado, sem novas dependências

## Valor de Negócio
Motion bem feito é o diferencial entre "app funcional" e "produto premium". É o que faz o usuário falar "esse app é bonito" para alguém.

## Riscos
- `prefers-reduced-motion` deve ser respeitado — envolver todos os keyframes em `@media (prefers-reduced-motion: no-preference)`
- Animações muito pesadas em dispositivos antigos — testar em hardware real ou throttling DevTools
