# Story 2.1: Design System Audit & Token Foundation

## Status
Done

## Descrição
Auditar todo o frontend atual para mapear inconsistências visuais (valores hardcoded, tokens não-canônicos, espaçamentos avulsos, tipografia inconsistente) e expandir o `@theme` de `src/index.css` com os tokens completos que servirão de fundação para todas as stories do epic 2.

## Critérios de Aceite
- [x] Relatório de auditoria gerado em `docs/design-system-audit.md` listando todas as inconsistências encontradas por categoria (cor, tipografia, espaçamento, sombra, border-radius, motion)
- [x] `src/index.css` expandido com tokens canônicos de: tipografia (size xs→4xl + weights + leading), border-radius (sm/md/lg/xl/2xl/full), elevation (shadow-sm/md/lg/gold/gold-strong) e motion (duration-fast/base/slow/slower/count + ease-out/in-out/decelerate)
- [x] Guia de componentes criado em `docs/component-guide.md` documentando: como usar cada token, padrão de card KPI, padrão de botão primário/secundário, padrão de input, padrão de lista de transações, empty state, modal, toast
- [x] Zero valores hex/px hardcoded introduzidos nos arquivos modificados
- [ ] `npm run lint` passa sem erros
