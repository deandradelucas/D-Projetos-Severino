# Story 7.1 — Excel Parser Backend

## Status
Done

## Descrição
Criar `server/lib/import/excel-parser.mjs` que recebe um Buffer de arquivo Excel (.xlsx, .xls) ou CSV, usa a biblioteca `xlsx` (SheetJS) para ler as linhas, envia as 5 primeiras linhas ao Gemini para detectar automaticamente quais colunas correspondem a data, descrição, valor e tipo (débito/crédito), e retorna um array normalizado de transações prontas para importação.

## Critérios de Aceite

### Instalação e Setup
- [x] `xlsx` adicionado ao `package.json` via `npm install xlsx`
- [x] Import via ESM: `import * as XLSX from 'xlsx'`

### Parsing do Arquivo
- [x] Aceita Buffer com mimeType: `application/vnd.openxmlformats-officedocument.spreadsheetml.sheet`, `application/vnd.ms-excel`, `text/csv`
- [x] Lê a primeira sheet do workbook
- [x] Converte para array de arrays (raw rows), preservando strings e números
- [x] Limite: rejeita arquivos com mais de 2.000 linhas (retorna erro claro)

### Detecção de Colunas via Gemini
- [x] Envia as primeiras 6 linhas (header + 5 dados) ao Gemini em prompt estruturado
- [x] Prompt solicita JSON `{col_data, col_descricao, col_valor, col_tipo, tipo_credito_label, tipo_debito_label}`
- [x] `col_*` são índices numéricos (0-based) da coluna correspondente
- [x] `col_tipo` pode ser `null` se o arquivo não tiver coluna de tipo explícita (ex.: Nubank usa valor negativo = débito)
- [x] Fallback: se Gemini falhar ou retornar schema inválido, tenta heurísticas de nomes de colunas (data/date, descrição/description, valor/amount/value, tipo/type/credit/debit)
- [x] Resultado do mapeamento é logado com `log.info`

### Normalização das Linhas
- [x] Para cada linha de dados (pula cabeçalho):
  - `data`: parse para ISO string `YYYY-MM-DD`; aceita DD/MM/YYYY, MM/DD/YYYY, DD-MM-YYYY, número serial Excel
  - `descricao`: string trim, max 255 chars
  - `valor`: número positivo (absoluteValue)
  - `tipo`: `'DESPESA'` ou `'RECEITA'`; se `col_tipo` não existe: valor negativo = DESPESA, positivo = RECEITA
- [x] Linhas com `data` inválida, `valor` zerado ou `descricao` vazia são ignoradas (não jogam erro)
- [x] Retorna `Array<{data: string, descricao: string, valor: number, tipo: 'DESPESA'|'RECEITA'}>`

### Tratamento de Erros
- [x] Arquivo não-Excel retorna `{ error: 'FORMATO_INVALIDO', message: '...' }`
- [x] Arquivo > 10MB retorna `{ error: 'ARQUIVO_MUITO_GRANDE', message: '...' }`
- [x] Sheet vazia retorna `{ error: 'PLANILHA_VAZIA', message: '...' }`
- [x] Zero linhas válidas após normalização retorna `{ error: 'NENHUMA_TRANSACAO', message: '...' }`

## Notas Técnicas
- SheetJS: `XLSX.read(buffer, { type: 'buffer', cellDates: false })` — manter como number/string para detectar formato de data
- Datas Excel: número serial (ex.: 45678) → converter com `XLSX.SSF.parse_date_code()` ou `new Date(0).setDate(serial - 25569)`
- Gemini model: `gemini-1.5-flash` (barato, suficiente para detecção de colunas)
- Max output tokens: 150 (só precisa do JSON de mapeamento)
