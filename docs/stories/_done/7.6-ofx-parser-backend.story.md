# Story 7.6 — OFX Parser Backend

## Status
Done

## Descrição
Criar `server/lib/import/ofx-parser.mjs` que parseia arquivos OFX/QFX exportados por bancos brasileiros (Bradesco, Itaú, Banco do Brasil, Caixa, Santander, Nubank, etc.). OFX v1.x usa SGML (não XML válido) — implementado com parser manual sem dependências externas. OFX v2.x usa XML e é parseado via `DOMParser`/regex. Retorna o mesmo formato normalizado das outras Stories.

## Critérios de Aceite

### Detecção de Versão
- [x] Lê os primeiros 500 bytes do buffer para detectar versão:
  - `OFXHEADER:100` → OFX v1.x (SGML)
  - `<?OFX` ou `<?xml` → OFX v2.x (XML)
  - Fallback: tenta SGML se nenhum header reconhecido

### Parser SGML (OFX v1.x — mais comum nos bancos BR)
- [x] Extrai todos os blocos `<STMTTRN>...</STMTTRN>` (ou sem fechamento — SGML não requer)
- [x] Para cada bloco, extrai via regex simples:
  - `<TRNTYPE>` → tipo da transação
  - `<DTPOSTED>` → data (YYYYMMDD ou YYYYMMDDHHmmss)
  - `<TRNAMT>` → valor (negativo = DESPESA, positivo = RECEITA)
  - `<MEMO>` → descrição preferencial
  - `<NAME>` → descrição alternativa (usa se MEMO estiver vazio)
  - `<FITID>` → ID único da transação (usado para deduplicação)
- [x] Função auxiliar `parseSgmlTag(block, tag)` — extrai valor de `<TAG>VALOR\n`

### Parser XML (OFX v2.x)
- [x] Usa regex para extrair `<STMTTRN>...</STMTTRN>` do XML
- [x] Mesmo mapeamento de campos da versão SGML
- [x] Compatível com QBO (QuickBooks Online — mesmo formato, extensão diferente)

### Mapeamento de TRNTYPE para tipo Severino
| OFX TRNTYPE | Severino tipo |
|---|---|
| `DEBIT`, `CHECK`, `PAYMENT`, `DIRECTDEBIT` | `DESPESA` |
| `CREDIT`, `DEP`, `DIRECTDEP`, `INTEREST`, `DIVIDEND` | `RECEITA` |
| `OTHER`, `FEE` | pelo sinal do TRNAMT (negativo=DESPESA) |

### Normalização de Data
- [x] YYYYMMDD → ISO `YYYY-MM-DD`
- [x] YYYYMMDDHHmmss → extrai só data `YYYY-MM-DD`
- [x] Data inválida → linha descartada silenciosamente

### Normalização de Valor
- [x] `TRNAMT` pode ser `"-87.50"`, `"87.50"`, `"-87,50"` — normaliza para float positivo
- [x] Valor 0 → linha descartada

### Deduplicação via FITID
- [x] `<FITID>` é o ID único da transação no banco emissor
- [x] Se presente: usa `FITID` como parte do hash de deduplicação (em vez de hash do conteúdo)
- [x] Hash: `sha256(${usuarioId}:ofx:${fitid})`
- [x] Se ausente: fallback para hash padrão de conteúdo (Story 7.3)

### Tratamento de Erros
- [x] Arquivo não-OFX → `{ error: 'FORMATO_INVALIDO' }`
- [x] Nenhuma `<STMTTRN>` encontrada → `{ error: 'NENHUMA_TRANSACAO' }`
- [x] Exceção de parsing → `{ error: 'FORMATO_INVALIDO', detail: mensagem }`

### Interface Pública
- [x] Função exportada: `async function parseOfxTransactions(buffer): Promise<ParsedRows | ErrorResult>`
- [x] Mesmo retorno das outras stories: `Array<{data, descricao, valor, tipo}>`
- [x] Sem dependências externas (parser zero-dep)

## Notas Técnicas

**Estrutura típica OFX v1.x de banco BR:**
```
OFXHEADER:100
DATA:OFXSGML
VERSION:102
SECURITY:NONE
ENCODING:UTF-8
CHARSET:1252
COMPRESSION:NONE
OLDFILEUID:NONE
NEWFILEUID:NONE

<OFX>
<STMTTRNRS>
<STMTRS>
<BANKTRANLIST>
<STMTTRN>
<TRNTYPE>DEBIT
<DTPOSTED>20240115
<TRNAMT>-87.50
<FITID>20240115001
<MEMO>MERCADO EXTRA - PAGTO DEBITO
</STMTTRN>
<STMTTRN>
<TRNTYPE>CREDIT
<DTPOSTED>20240120
<TRNAMT>3500.00
<FITID>20240120001
<MEMO>SALARIO EMPRESA XYZ
</STMTTRN>
</BANKTRANLIST>
</STMTRS>
</STMTTRNRS>
</OFX>
```

**Regex para blocos SGML:**
```js
const blockRe = /<STMTTRN>([\s\S]*?)(?=<STMTTRN>|<\/BANKTRANLIST>|<\/STMTTRNRS>|$)/gi
const tagRe = (tag) => new RegExp(`<${tag}>([^\\n\\r<]+)`, 'i')
```

**Encoding:** Bancos BR às vezes exportam Latin-1/ISO-8859-1. Se houver caracteres estranhos, converter com `Buffer.toString('latin1')` como fallback.

**FITID como dedup:** É o campo mais confiável para deduplicação — é o ID do banco emissor, único por conta+transação. Muito melhor que hash de conteúdo porque evitar duplicatas mesmo se o usuário editar a descrição e reimportar.
