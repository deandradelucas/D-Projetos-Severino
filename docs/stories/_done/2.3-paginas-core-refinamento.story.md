# Story 2.3: Páginas Core — Refinamento Visual

## Status
Done

## Descrição
Aplicar o design system definido na Story 2.1 em todas as páginas de conteúdo. O objetivo é consistência total: o usuário deve sentir que cada tela pertence ao mesmo produto premium. Foco nas páginas de alto uso diário (Dashboard, Transações, Relatórios) e páginas de primeira impressão (Login, Cadastro, Pagamento).

## Critérios de Aceite

### Dashboard
- [x] Cards KPI com elevação consistente (`--shadow-gold` no card principal de saldo)
- [x] Saldo exibido com tipografia `--font-size-2xl` ou maior, peso bold
- [x] Lista de transações recentes com separador sutil (`--color-border`) e ícone de categoria padronizado
- [x] Empty state ilustrado e encorajador quando não há transações
- [x] Skeleton screens idênticos ao layout final (mesmo tamanho de cada elemento)

### Transações
- [x] Cabeçalho de filtros padronizado com chips usando tokens (border, bg, texto)
- [x] Linha de transação: ícone + nome + categoria + valor alinhados consistentemente
- [x] Valores positivos em `--color-success`, negativos em `--color-error`
- [x] "Carregar mais" como botão ghost estilizado, não link simples
- [x] Empty state com mensagem clara e CTA para nova transação

### Relatórios
- [x] Gráficos com cores usando tokens da paleta (accent-gold para receita, error para despesa)
- [x] Cards de resumo com tipografia e espaçamento consistentes
- [x] Seletor de período padronizado com estilo do design system

### Agenda
- [x] Itens de agenda com mesmo padrão visual da lista de transações
- [x] Status de evento com badge usando tokens de cor

### Login / Cadastro
- [x] Formulários com inputs padronizados: border `--color-border`, focus ring `--color-accent-gold`
- [x] Botão primário com efeito gold e border-radius consistente
- [x] Logotipo/marca centralizado com tamanho adequado para splash de PWA

### Configurações
- [x] Seções com separador e títulos em `--color-text-secondary`
- [x] Toggle de tema com animação suave
- [x] Items de config com padding consistente e hover state

### Pagamento / BemVindoAssinatura
- [x] Hierarquia visual clara: preço dominante, benefícios legíveis
- [x] CTA principal em gold com hover animado
