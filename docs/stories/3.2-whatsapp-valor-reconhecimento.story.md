# Story 3.2: Melhoria no Reconhecimento de Valores em Áudios WhatsApp

## Status
InReview

## Descrição
O sistema falha ao interpretar valores falados ou escritos em formato brasileiro quando recebidos via WhatsApp. Dois bugs críticos: (1) `extrairValorBasicoFromTexto` interpreta "2.000" como 2.0 em vez de 2000 por má interpretação do separador de milhar BR; (2) valores por extenso como "dois mil" ou "três mil e quinhentos reais" não são parseados pelo fallback local. Além disso, o prompt de IA e a instrução de transcrição de áudio não orientam o Gemini explicitamente sobre esses formatos.

## Valor de Negócio
Erros de valor em transações financeiras são críticos — um usuário que fala "dois mil reais" e o sistema cadastra R$ 2,00 perde confiança imediata no produto. Fix de alta prioridade para qualidade de dados.

## Arquitetura de Implementação

### `parseBRVerbalValor(texto)` — Estratégia

```
Unidades:   um=1, dois=2, três=3, quatro=4, cinco=5, seis=6, sete=7, oito=8, nove=9
Dezenas:    dez=10, onze=11, doze=12, treze=13, quatorze=14, quinze=15,
            dezesseis=16, dezessete=17, dezoito=18, dezenove=19,
            vinte=20, trinta=30, quarenta=40, cinquenta=50, sessenta=60,
            setenta=70, oitenta=80, noventa=90
Centenas:   cem=100, cento=100, duzentos=200, trezentos=300, quatrocentos=400,
            quinhentos=500, seiscentos=600, setecentos=700, oitocentos=800,
            novecentos=900
Multiplicador: mil=1000

Algoritmo:
  1. Normaliza texto (lowercase, remove "reais", "real", "uns", "umas", "de", "e")
  2. Detecta presença de "mil" no texto
  3. Se "mil":
     - Parte antes de "mil" = centenas/dezenas/unidades antes do multiplicador (ou 1 se vazio)
     - Parte depois de "mil" = centenas/dezenas/unidades adicionais
     - valor = (parte_antes * 1000) + parte_depois
  4. Se não "mil": soma centenas + dezenas + unidades
  5. Retorna valor se > 0, null caso contrário
```

### `extrairValorBasicoFromTexto(message)` — Regex corrigido

```js
// Novo regex que captura formatos BR completos:
// "2.000,50" | "2.000" | "89,90" | "50"
const brNumRegex = /(\d{1,3}(?:\.\d{3})+(?:,\d{1,2})?|\d+(?:,\d{1,2})?)/

// Normalização:
// Se tem ponto E vírgula: remove pontos (milhar), troca vírgula por ponto
// Se tem ponto SEM vírgula E 3 dígitos após ponto: é milhar (remove ponto)
// Se tem vírgula SEM ponto: troca vírgula por ponto (decimal)
// Caso contrário: deixa como está
```
